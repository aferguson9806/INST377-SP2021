Takeaways: Python and JavaScript

In conclusion, while both Python and JavaScript have their strengths and unique features, if I were to be developing a dynamic visual application, I would use JavaScript.

However, when it comes to data science and big data analysis, I would absolutely use Python and its library Pandas.

Ultimately, most languages are capable of doing most things a programmer might want to do and it comes down to personal preference and what language is best suited for your existing skillset. Personally, I'm not great with tkinter, and therefore I don't often have a desire to work on GUIs in Python. I could much better spend my time and effort on a web application using HTML, CSS, and JavaScript.

Language Ecosystem Considerations

When it comes to choosing between Python and JavaScript and which language to learn, JavaScript and Python are both so common and prominent that you can trust that you'll be able to find a job just about anywhere if you know one or the other, certainly if you know both of them.

According to data from GitHut, the two most common languages for activity (defined as pull requests, pushes, stars, and issues) on GitHub are JavaScript and Python.

One consideration I think is worth making is that Python is famously open-source, even for commercial use. Because of this, there are over 130,000 libraries available in Python. If you can't find a library for what you need, you're 100% allowed to write your own and distribute it!
